Project Sentinel est une pile anti-triche axée Windows qui protège les clients de jeu via un SDK in-process léger, un service usermode signé distinct et un pilote en mode noyau. Le SDK expose une petite API Init/Heartbeat/Shutdown, mappe automatiquement le .text PE et les imports pour le hachage d'intégrité sans offsets manuels, et communique avec le service via named pipes ou ALPC afin que l'application des règles reste hors du binaire du jeu. L'agent noyau utilise des callbacks documentés tels que ObRegisterCallbacks et les routines de notification de chargement d'image pour la protection de processus et la détection de pilotes non autorisés, tandis que les composants usermode ajoutent la validation de pile d'appels et des gardes d'overlay ou de station de fenêtres. Un backend Node.js ingère la télémétrie via mTLS, stocke les signatures (par exemple dans Postgres) et diffuse les mises à jour de liste noire via SSE pour une détection et une réponse pilotées par les analystes. Il s'adresse aux studios qui construisent de l'anti-triche client, du monitoring d'intégrité et la distribution de signatures de triche, plutôt qu'à un produit commercial fini.
