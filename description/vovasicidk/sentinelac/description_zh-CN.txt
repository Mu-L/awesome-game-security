Project Sentinel 是一套以 Windows 为首的反作弊技术栈，通过精简的进程内 SDK、独立的已签名用户态服务以及内核态驱动来保护游戏客户端。SDK 提供简洁的 Init/Heartbeat/Shutdown API，可自动映射 PE 的 .text 段与导入表以进行完整性哈希而无需手动偏移，并通过命名管道或 ALPC 与服务通信，使执法逻辑留在游戏二进制之外。内核代理使用 ObRegisterCallbacks、映像加载通知等已文档化的回调实现进程保护与未授权驱动检测，用户态组件则补充调用栈校验以及覆盖层或窗口站防护。Node.js 后端通过 mTLS 接收遥测数据，将签名存储（例如在 Postgres 中），并通过 SSE 分发黑名单更新，以支持分析人员驱动的检测与响应。它面向需要构建客户端反作弊、完整性监控与作弊签名分发的游戏工作室，而非作为成品商业产品。
