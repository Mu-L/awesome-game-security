Project Sentinel — это Windows-first античит-стек, защищающий игровые клиенты тонким in-process SDK, отдельным подписанным usermode-сервисом и драйвером в режиме ядра. SDK предоставляет небольшой API Init/Heartbeat/Shutdown, автоматически отображает PE .text и импорты для хэширования целостности без ручных смещений и общается с сервисом через named pipes или ALPC, чтобы enforcement оставался вне игрового бинарника. Kernel-агент использует документированные колбэки вроде ObRegisterCallbacks и load-image notify для защиты процессов и обнаружения несанкционированных драйверов, а usermode-компоненты добавляют проверку call stack и защиту от overlay или window station. Node.js-бэкенд принимает телеметрию по mTLS, хранит сигнатуры (например в Postgres) и рассылает обновления blacklist через SSE для detection and response под управлением аналитиков. Он рассчитан на студии, строящие клиентский античит, мониторинг целостности и распространение cheat-сигнатур, а не как готовый коммерческий продукт.
