Project Sentinel es una pila anti-trampas centrada en Windows que protege los clientes de juego con un SDK in-process ligero, un servicio usermode firmado independiente y un controlador en modo kernel. El SDK expone una API pequeña Init/Heartbeat/Shutdown, mapea automáticamente el .text PE y los imports para hashing de integridad sin offsets manuales, y habla con el servicio mediante named pipes o ALPC para que la aplicación de políticas quede fuera del binario del juego. El agente de kernel usa callbacks documentados como ObRegisterCallbacks y rutinas de notificación de carga de imagen para protección de procesos y detección de controladores no autorizados, mientras los componentes usermode añaden validación de call stack y guards de overlay o window station. Un backend Node.js ingiere telemetría por mTLS, almacena firmas (por ejemplo en Postgres) y distribuye actualizaciones de lista negra vía SSE para detección y respuesta impulsadas por analistas. Está pensado para estudios que construyen anti-trampas de cliente, monitorización de integridad y distribución de firmas de trampas, no como un producto comercial acabado.
