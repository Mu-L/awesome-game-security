Project Sentinel è uno stack anti-cheat Windows-first che protegge i client di gioco con un SDK in-process snello, un servizio usermode firmato separato e un driver in modalità kernel. L'SDK espone una piccola API Init/Heartbeat/Shutdown, mappa automaticamente il .text PE e gli import per l'hashing di integrità senza offset manuali e comunica con il servizio tramite named pipes o ALPC, così l'enforcement resta fuori dal binario del gioco. L'agente kernel usa callback documentati come ObRegisterCallbacks e routine di notifica load-image per la protezione dei processi e il rilevamento di driver non autorizzati, mentre i componenti usermode aggiungono validazione dello call stack e guard di overlay o window station. Un backend Node.js ingerisce telemetria via mTLS, memorizza le signature (ad esempio in Postgres) e propaga aggiornamenti della blacklist via SSE per detection e response guidate dagli analisti. È pensato per studi che costruiscono anti-cheat client, monitoraggio dell'integrità e distribuzione di signature cheat, non come prodotto commerciale finito.
