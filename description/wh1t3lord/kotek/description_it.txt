Kotek è un framework core modulare in C/C++ che costituisce lo strato di fondazione di uno stack di game engine a tre livelli, fornendo servizi sostituibili su cui possono basarsi il codice di engine e di gioco di livello superiore. È organizzato in molti piccoli moduli CMake con interfacce ktkI* stabili registrate tramite un manager principale, così i singoli backend per matematica, container, logging, filesystem, windowing, UI, ECS e rendering possono essere scambiati in fase di configurazione o sovrascritti a runtime tramite plugin. I backend supportati includono opzioni come GLM o DirectXMath, GLFW, Dear ImGui, RmlUi, CEF, pico_ecs e bgfx, insieme a implementazioni proprie senza dipendenze ove praticabile. Le build mirano prima a Windows con astrazioni OS per Linux e supportano modalità di linkage statico, shared ed esplicito a plugin. Il caso d’uso principale è lo sviluppo di giochi e engine dove serve un core runtime C++ personalizzabile e con dipendenze commutabili.
