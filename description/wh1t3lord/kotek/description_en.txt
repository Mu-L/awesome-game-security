Kotek is a modular C/C++ core framework that forms the foundation layer of a three-layer game engine stack, providing replaceable services that higher-level engine and game code can build on. It is organized as many small CMake modules with stable ktkI* interfaces registered through a main manager, so individual backends for math, containers, logging, filesystem, windowing, UI, ECS, and rendering can be swapped at configure time or overridden at runtime via plugins. Supported backends include options such as GLM or DirectXMath, GLFW, Dear ImGui, RmlUi, CEF, pico_ecs, and bgfx, along with no-dependency own implementations where practical. Builds target Windows first with Linux OS abstractions, and support static, shared, and explicit plugin linkage modes. The primary use case is game and engine development where a customizable, dependency-switchable C++ runtime core is needed.
